ปัญหาที่ต้องแก้ — Shortest Path vs MST

Shortest Path — หาเส้นทางสั้นสุด
จากจุดเริ่มต้น 1 จุด → ไปยังทุกจุดอื่น ด้วยค่าน้ำหนักรวมน้อยที่สุด

ใช้: GPS นำทาง, routing packet ในเครือข่าย
MST — ต้นไม้ครอบคลุมน้ำหนักน้อยสุด
เชื่อมทุก vertex เข้าด้วยกัน โดยใช้ edge น้อยที่สุดและรวมน้ำหนักน้อยที่สุด

ใช้: วางสายไฟ, เครือข่าย internet ราคาถูก
จำง่าย:
Shortest Path = "ฉันจะไป BKK จาก CNX ยังไงให้เร็วสุด?" — สนใจ path จาก source ไป destination
MST = "จะลากสายไฟให้ครบทุกบ้านโดยใช้สายน้อยสุด?" — สนใจเชื่อมทุกจุดรวมกันให้ถูกที่สุด

Dijkstra's Algorithm — หาเส้นทางสั้นสุด (Lab 1)
intuition — เข้าใจก่อนจำ
Dijkstra = เหมือน ระลอกน้ำ แต่ชาญฉลาดกว่า BFT เพราะคำนึงถึง น้ำหนัก
แนวคิด: เยี่ยม vertex ที่ "ถูกที่สุด/ใกล้ที่สุด" จาก source ก่อนเสมอ → ค่อย ๆ ขยายออก
ใช้ Priority Queue (Min-Heap) — ดึงตัวที่มี cost น้อยสุดออกมาก่อน
ข้อจำกัดสำคัญ: ใช้ได้กับ graph ที่ น้ำหนักไม่ติดลบ เท่านั้น!
เปรียบเทียบ: ถ้า BFT = นับจำนวน hop, Dijkstra = นับน้ำหนักรวม
ทางที่ hop น้อยกว่า ≠ ทางที่ถูกกว่า เสมอ — Dijkstra จึงเลือกทางที่ถูกสุดเสมอ
ออกสอบ — ขั้นตอน Dijkstra จำให้ได้
1. dist[src] = 0, ทุกตัวอื่น = ∞
→
2. ใส่ (0, src) ลง Priority Queue
→
3. ดึงตัวที่ cost น้อยสุดออก
→
4. Relax neighbor ทุกตัว (อัพเดต dist ถ้าถูกกว่า)
→
5. ทำซ้ำจนหมด PQ
Relaxation คือหัวใจ: ถ้า dist[u] + weight(u,v) < dist[v] → อัพเดต dist[v]
Dijkstra — Python implementation
import heapq

def dijkstra(graph, src):
dist = {v: float('inf') for v in graph} # เริ่มต้น ∞ ทุกตัว
dist[src] = 0
pq = [(0, src)] # (cost, vertex) → min-heap

while pq:
cost, u = heapq.heappop(pq) # ดึงตัวที่ถูกที่สุด
if cost > dist[u]: continue # outdated entry → ข้ามไป

for v, w in graph[u].items(): # ไล่ neighbor
new_cost = dist[u] + w
if new_cost < dist[v]: # Relaxation!
dist[v] = new_cost
heapq.heappush(pq, (new_cost, v))

return dist

Minimum Spanning Tree (MST) — เชื่อมทุกจุดให้ถูกสุด
concept — MST คืออะไร
Spanning Tree = tree ที่เชื่อมทุก vertex ใน graph โดยมี edge จำนวน V-1 เส้น (V คือจำนวน vertex)
MST = Spanning Tree ที่ผลรวมน้ำหนัก edge น้อยที่สุด ในบรรดา Spanning Tree ทั้งหมด
MST ไม่มี cycle — เพราะถ้ามี cycle แสดงว่ามี edge เกิน จะตัดออกได้เสมอ
G4 มี 9 vertex → MST ต้องมี edge ครบ 8 เส้นพอดี
ทำไมต้อง V-1? เพราะ tree ที่เชื่อม n node ต้องมี n-1 edge — น้อยกว่านี้ไม่ connected มากกว่านี้มี cycle
ออกสอบ — 2 algorithm สำหรับ MST ต้องเปรียบเทียบได้
Prim-Jarnik — โตจากจุดเดียว
เริ่มจาก vertex ใด ๆ → ขยาย MST ทีละ 1 vertex โดยเลือก edge ที่ถูกที่สุดที่เชื่อมเข้า MST
Kruskal's — เรียง edge แล้วเลือก
เรียง edge ทุกเส้นจากถูก→แพง → เลือกทีละเส้น ถ้าไม่สร้าง cycle ก็เพิ่ม ไม่สร้างก็ข้าม
ทั้งสองได้ MST เดียวกัน (total weight เท่ากัน) แต่ลำดับเลือก edge อาจต่างกัน
Prim = เหมือนปลูกต้นไม้จากรากเดียว | Kruskal = เหมือนเลือกอิฐราคาถูกสุดก่อน

Prim-Jarnik Algorithm — ขยาย MST จากจุดเดิม (Lab 2)
intuition
Prim = คล้าย Dijkstra มาก — ต่างกันที่ key ของ PQ คือ น้ำหนัก edge ที่เชื่อมเข้า MST ไม่ใช่ระยะสะสม
เริ่มจาก vertex ใด ๆ → ทุกรอบ เลือก vertex ข้างนอกที่ ใกล้ MST ที่สุด มาเพิ่ม
ใช้ Priority Queue เช่นกัน แต่เก็บ (edge_weight, neighbor_vertex) ไม่ใช่ (total_dist, vertex)
ออกสอบ — ขั้นตอน Prim-Jarnik
1. เริ่มจาก vertex ใด ๆ ก็ได้ → ใส่ neighbor ทุกตัวลง PQ
→
2. ดึง (weight, v) ที่น้อยสุดออก
→
3. ถ้า v ยังไม่อยู่ใน MST → เพิ่ม edge นี้ลง MST
→
4. ใส่ neighbor ของ v ที่ยังไม่อยู่ใน MST ลง PQ
→
5. ทำซ้ำจน MST มีครบ V-1 edge
Prim-Jarnik — Python implementation
import heapq

def MST_PrimJarnik(graph):
start = next(iter(graph)) # เริ่มจาก vertex แรก
in_mst = {start}
mst_edges = []
pq = []

for neighbor, w in graph[start].items():
heapq.heappush(pq, (w, start, neighbor))

while pq and len(in_mst) < len(graph):
w, u, v = heapq.heappop(pq) # edge ถูกสุด
if v in in_mst: continue # v อยู่ใน MST แล้ว → ข้าม
in_mst.add(v)
mst_edges.append((u, v, w)) # เพิ่ม edge นี้ใน MST
for neighbor, w2 in graph[v].items():
if neighbor not in in_mst:
heapq.heappush(pq, (w2, v, neighbor))

return mst_edges

Kruskal's Algorithm — เรียง Edge แล้วเลือก (Lab 3)
intuition
Kruskal = ต่างจาก Prim ตรงที่ไม่ขยายจากจุดเดิม — มองภาพรวม edge ทั้งหมดก่อน
เรียง edge ทุกเส้นจากถูก→แพง → ไล่เลือกทีละเส้น → เพิ่มถ้า ไม่ทำให้เกิด cycle
ใช้โครงสร้าง Union-Find (Disjoint Set) — ตรวจว่าสอง vertex อยู่ใน component เดียวกันหรือเปล่า
Union-Find คืออะไร? เป็นโครงสร้างที่ตอบคำถาม "A กับ B เชื่อมกันอยู่ไหม?" ได้เร็วมาก
ถ้า A กับ B อยู่ใน component เดียวกัน → เพิ่ม edge A-B จะทำให้เกิด cycle → ข้าม!
ออกสอบ — ขั้นตอน Kruskal
1. รวม edge ทุกเส้นในกราฟ
→
2. เรียงจากน้ำหนักน้อย → มาก
→
3. ไล่ edge ทีละเส้น
→
4. ถ้า u, v ต่าง component → เพิ่มลง MST + Union(u, v)
→
5. ถ้า u, v component เดียวกัน → ข้าม (จะเกิด cycle)
Kruskal — Python implementation
def MST_Kruskal(graph):
# 1. รวม edge ทั้งหมด (undirected → ไม่นับซ้ำ)
edges = []
seen = set()
for u in graph:
for v, w in graph[u].items():
if (v, u) not in seen:
edges.append((w, u, v)); seen.add((u, v))
edges.sort() # เรียงน้ำหนักจากน้อยไปมาก

# 2. Union-Find
parent = {v: v for v in graph}
def find(x): return x if parent[x]==x else find(parent[x])
def union(a, b): parent[find(a)] = find(b)

# 3. เลือก edge
mst = []
for w, u, v in edges:
if find(u) != find(v): # ต่าง component → ปลอดภัย
mst.append((u, v, w))
union(u, v)
return mst
