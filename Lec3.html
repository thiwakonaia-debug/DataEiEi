Hashing คืออะไร — แนวคิดหลัก
concept
Hashing = เทคนิคแปลง "ค่าข้อมูล" ให้เป็น "ตำแหน่งที่อยู่" โดยตรง ไม่ต้องไล่เปรียบเทียบทีละตัว
Hash Table = โครงสร้างข้อมูลที่ใช้ร่วมกับ hashing ประกอบด้วย 2 สดมภ์: Hash Value (ที่อยู่) + Hash Key (ค่าข้อมูล)
ขนาดของ Hash Table = ความยาวของ array ที่ใช้เก็บ
เปรียบเสมือนตู้เก็บเอกสารที่มีหมายเลขล็อค — ชื่อเอกสาร (key) → คำนวณหมายเลขล็อค (hash) → เปิดล็อคตรงนั้นได้เลย ไม่ต้องไล่เปิดทุกล็อค
ออกสอบ — Complexity ต้องจำ
Time Complexity
Insert, Find, Delete = O(1) เร็วที่สุดในบรรดาโครงสร้างข้อมูลทั้งหมด
Space Complexity
S(n) = O(n) ใช้พื้นที่สัดส่วนกับจำนวนข้อมูล
เปรียบกับ Array: O(n) search, Balanced Tree: O(log n) search — Hashing ชนะทั้งคู่ด้วย O(1)
ข้อดี vs ข้อเสีย
ข้อดี
Insert, Find, Delete รวดเร็ว O(1) — เหมาะกับข้อมูลจำนวนมากและการทำดัชนี
ข้อเสีย
ไม่เหมาะกับ Range Search (>10 <20) ไม่รองรับเรียงลำดับ และถ้า hash function ไม่ดีจะเกิด collision มาก

ฟังก์ชันแฮช (Hash Function) — 4 วิธี
concept
Hash Function h แปลง k (Hash Key = ค่าข้อมูล) → h(k) (Hash Value = ตำแหน่งที่อยู่)
เป็น value-to-address transformation — ค่าเดิม key เดิมต้องได้ hash value เดิมเสมอ
k (Hash Key)
→
h (Hash Function)
→
h(k) (Hash Value)
ออกสอบ — วิธีที่ 1: Modulo (นิยมมากที่สุด)
h(k) = k % M
k = กุญแจแฮช, M = ขนาดของตารางแฮช, % = หารเอาเศษ
M ควรเป็น จำนวนเฉพาะ (prime number) หรือ 2ⁿ เพื่อลด collision
ตัวอย่าง: k=87654, M=100 → h(k) = 87654 % 100 = 54
ออกสอบ — วิธีที่ 2: Digit Folding
ขั้นตอน: แบ่ง key เป็นส่วน r หลัก → รวมทุกส่วน → ตัดเลขทด (last carry) ออก
k = 6624410909, r = 2 → แบ่งเป็น 66, 24, 41, 09, 09
รวม: 66+24+41+09+09 = 149 → ตัด carry ออก → h(k) = 49

ตัวอย่างสอบ: k=87654, r=2 → 87, 65, 4 → 87+65+4 = 156 → h(k) = 56
ออกสอบ — วิธีที่ 3: Mid-Square
ขั้นตอน: ยก key กำลัง 2 → เลือก r หลักตรงกลางของผลลัพธ์
k = 60 → 60² = 3600 → ตรงกลาง 2 หลัก → h(k) = 60

ตัวอย่างสอบ: k=87654, r=2 → 87654² = 7683215716 → ตรงกลาง 2 หลัก → h(k) = 21
ออกสอบ — วิธีที่ 4: Multiplication Method
สูตร: h(k) = floor(M × (kA mod 1)) เมื่อ 0 < A < 1
ขั้นตอน: (1) คูณ k×A → (2) เอาเฉพาะทศนิยม → (3) คูณ M → (4) floor
k=12345, A=0.357840, M=100
kA = 4417.5348 → ทศนิยม = 0.5348 → ×100 = 53.48 → floor → h(k) = 53

Non-Numeric Keys — แฮช String อย่างไร
concept
ถ้า key เป็น string เช่น "CAT" ต้องแปลงเป็นตัวเลขก่อน จึงค่อยใส่ hash function
วิธีแปลง: ใช้ ASCII (A=65, B=66, ..., Z=90) หรือกำหนดเอง (A=1, B=2, ...)
ออกสอบ — 3 วิธีแปลง String
วิธี 1: Adding Digits
รวม code ทุกตัวอักษร
CAT (A=1..Z=26):
3+1+19 = 23
วิธี 2: Multiplying by Powers
คูณด้วยฐาน 27
CAT: 3×27²+1×27¹+19×27⁰ = 2233
วิธี 3: Multiplying by Indices
คูณด้วย index ตำแหน่ง
CAT: 3×1+1×2+19×3 = 62
สังเกต: วิธี Adding Digits ไม่แยกความแตกต่างระหว่าง CAT กับ TAC (ผลรวมเท่ากัน) — วิธี 2 และ 3 แก้ปัญหานี้ได้
ตัวอย่าง Python (ASCII)
# แปลง "CAT" ด้วย ASCII แล้ว hash ด้วย modulo
sum_ascii = ord('C') + ord('A') + ord('T') # 67+65+84 = 216
h = sum_ascii % 7 # 216 % 7 = 6
print(h) # print 6

Collision — ปัญหาการชนกัน และการแก้ไข
ออกสอบ — Collision คืออะไร
Collision = เมื่อ key ต่างกัน แต่ฟังก์ชันแฮชให้ค่าเดียวกัน (ชนกันในช่องเดิม)
key เดิม → hash เดิมเสมอ แต่ key ต่างกัน อาจชนได้
ตัวอย่าง: h(k) = k % 7, ใส่ 7, 14, 21, 28, 35 → ทุกตัวได้ h(k) = 0 หมด → ชนทั้งหมด!
ออกสอบ — 2 แนวทางแก้ Collision
Separate Chaining (Open Hashing)
แต่ละช่องในตารางเก็บ linked list — ถ้าชนกันก็ต่อท้าย list ในช่องนั้น ง่ายแต่ใช้ memory พิเศษ
Open Addressing (Closed Hashing)
เก็บทุกอย่างในตารางเดียว — ถ้าชนให้หาช่องว่างใกล้เคียงตามกฎที่กำหนด
ออกสอบ — Open Addressing 3 วิธี
Linear Probing
เลื่อนไปทีละ 1 ช่อง
(h(k)+i) % M
ง่าย แต่เกิด clustering
Quadratic Probing
เลื่อนไปตาม i²
(h(k)+i²) % M
กระจายดีกว่า linear
Double Hashing
ใช้ h2 เมื่อชน
(h1+j×h2) % M
กระจายดีที่สุด
ตัวอย่าง Linear: h(k)=k%7, insert 12,19,33,20
12→slot 5, 19→slot 6, 33→slot 5 ชน! → (5+1)%7=6 ชนอีก! → (5+2)%7=0 ว่าง → ใส่ที่ 0
20→slot 6 ชน → (6+1)%7=0 ชน → (6+2)%7=1 ว่าง → ใส่ที่ 1
Double Hashing สูตรครบ
h(k, j) = (h1(k) + j × h2(k)) % M
j = จำนวนครั้งที่ชน (เริ่มที่ 0), h1 ใช้ปกติ, h2 ใช้เมื่อชนครั้งแรก
h1(k) = k%7, h2(k) = 1+(k%7)
ถ้า key=33 ชนที่ j=1: h(33,1) = (33%7 + 1×(1+33%7)) % 7 = (5+6)%7 = 4
5
Load Factor — ตัวชี้วัดประสิทธิภาพ
ออกสอบ — สูตรและการตีความ
Load Factor (λ) = จำนวน key ที่บรรจุ / ขนาดของตาราง
λ → 1 หมายถึงตารางแฮชใกล้เต็ม → โอกาสชนมากขึ้น → ประสิทธิภาพลดลง
เมื่อ λ ≈ 0.7 หรือ 0.75 ควรขยายตาราง ~2 เท่า แล้ว rehash ทุก key ใหม่
ตาราง size=7, ใส่ข้อมูล 5 ตัว → λ = 5/7 ≈ 0.71 → ถึงเวลา rehash!
ขยายเป็น size=13 (prime) → hash function เปลี่ยนเป็น k%13 → rehash ทุกตัวใหม่

การประยุกต์ใช้งาน — Real-world Hashing
สำคัญ — One-way Function
ฟังก์ชันแฮชเป็น ฟังก์ชันทางเดียว (one-way) — แปลงจาก key → hash value ได้ แต่ ย้อนกลับไม่ได้
นี่คือเหตุผลที่ hashing ใช้ในระบบความปลอดภัยได้ดี
3 การประยุกต์หลัก
🔐 Password Verification
เก็บ hash ของรหัสผ่าน ไม่เก็บรหัสตรง ๆ — ต้อง brute-force ถึงจะ hack ได้
📦 Checksums
ตรวจสอบว่าไฟล์ที่รับมาครบถ้วนไม่มีการเปลี่ยนแปลงระหว่างทาง (MD5, SHA1)
⛓️ Blockchain
ทุก block เก็บ hash ของตัวเองและ block ก่อนหน้า — ทำให้แก้ไขข้อมูลย้อนหลังไม่ได้
