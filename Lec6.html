ทำไมต้อง Sort — และ Swap คืออะไร
concept
Sorting = การจัดเรียงข้อมูลให้อยู่ในลำดับที่กำหนด (น้อยไปมาก หรือมากไปน้อย)
หัวใจของทุก sort คือ 2 operation: Comparison (เปรียบเทียบ) และ Swap (สับเปลี่ยน)
การวัดประสิทธิภาพ sort = นับ จำนวน comparison + swap ที่เกิดขึ้น
ตัวอย่าง array ที่ใช้ตลอด lecture นี้:
เริ่มต้น: [45, 23, 87, 12, 32, 4] → เป้าหมาย: [4, 12, 23, 32, 45, 87]
Swap ใน Python — จำให้ได้
ภาษาอื่นต้องใช้ temp variable แต่ Python ทำได้ 1 บรรทัด
# วิธีทั่วไป (ต้องใช้ temp)
temp = A[i]
A[i] = A[j]
A[j] = temp

# Python shortcut — ใช้บรรทัดเดียว!
A[i], A[j] = A[j], A[i]

Bubble Sort — เรียงแบบฟอง ไหลขึ้นทีละฟอง
intuition — เข้าใจก่อนจำ
Bubble Sort = ค่าที่มากที่สุดจะ "ลอย" ไปสุดขวาทุก round เหมือนฟองอากาศลอยขึ้น
ทุกรอบ เปรียบเทียบคู่ข้างๆ กัน → swap ถ้าซ้าย > ขวา → ขยับขวา → ทำซ้ำ
จบ 1 รอบ = ตัวใหญ่สุดไปนอนสุดขวาแล้ว → รอบต่อไปไม่ต้องสนมันอีก
จำง่าย: "ฟองใหญ่ลอยขึ้นเร็ว" — ทุก round ตัวใหญ่สุดใน unsorted ไปอยู่ที่ถูกต้องเสมอ

ออกสอบ — ขั้นตอน Bubble Sort
1. เริ่มซ้ายสุด
→
2. เปรียบเทียบ A[i] กับ A[i+1]
→
3. ถ้าซ้าย > ขวา → Swap
→
4. เลื่อนขวา 1 ช่อง
→
5. ทำซ้ำจนถึงปลาย (ลดลงทุกรอบ)
Bubble Sort — Python
def bubble_sort(A):
n = len(A)
for i in range(n - 1): # n-1 รอบ
for j in range(n - 1 - i): # ลดขอบขวาทุกรอบ
if A[j] > A[j+1]:
A[j], A[j+1] = A[j+1], A[j] # Swap!
return A

# Optimized: หยุดถ้าไม่มี swap เลย (Best Case O(n))
def bubble_sort_optimized(A):
n = len(A)
for i in range(n - 1):
swapped = False
for j in range(n - 1 - i):
if A[j] > A[j+1]:
A[j], A[j+1] = A[j+1], A[j]
swapped = True
if not swapped: break # ไม่มี swap = เรียงแล้ว!
return A
เพิ่ม swapped flag → ถ้าทั้งรอบไม่มี swap เลย = array เรียงแล้ว → หยุดได้ทันที
นี่คือเหตุผลที่ Bubble Sort Best Case = Ω(n) (ถ้าใช้ optimized version)

Selection Sort — หาตัวเล็กสุด แล้ว Swap ครั้งเดียว
intuition — เข้าใจก่อนจำ
Selection Sort = เหมือนจัดแถวนักเรียนตามส่วนสูง — วนดูทั้งแถว หาตัวเตี้ยสุด แล้ว swap มาหน้าสุด
ทุก round หา minimum ของ unsorted part → swap มาวางที่ตำแหน่งถูกต้อง → ส่วน sorted โตขึ้น 1
ข้อดีพิเศษ: Swap น้อยมาก — แต่ละรอบ swap ได้แค่ 1 ครั้ง = ทั้งหมดไม่เกิน n-1 ครั้ง
จำง่าย: "คัดเลือกเตี้ยสุด แล้วจัดหน้า" — ต่างจาก Bubble ที่ swap ได้หลายครั้งต่อรอบ

Selection Sort — Python
def selection_sort(A):
n = len(A)
for i in range(n - 1): # i = จุดเริ่มต้นของ unsorted
min_idx = i # สมมติว่า i คือ minimum ก่อน
for j in range(i + 1, n): # ไล่หา minimum จริง
if A[j] < A[min_idx]:
min_idx = j
if min_idx != i: # swap ก็ต่อเมื่อ min ไม่ใช่ตัวเอง
A[i], A[min_idx] = A[min_idx], A[i]
return A
สังเกต: Loop ใน (หา min) ทำงานทุกครั้ง แต่ Swap ทำเฉพาะตอนจำเป็น
→ Comparisons = เยอะเหมือนเดิม O(n²) แต่ Swaps = น้อยมาก ≤ n-1 ครั้ง

Insertion Sort — หยิบมาแทรก เหมือนจัดไพ่
intuition — เข้าใจก่อนจำ
Insertion Sort = เหมือนเล่นไพ่ — หยิบไพ่ใหม่มา 1 ใบ แล้วสอดแทรกเข้าไปในมือที่เรียงอยู่แล้ว
แบ่งเป็น 2 ส่วน: Sorted list (ซ้าย) และ Unsorted list (ขวา) — เริ่มต้น sorted มีแค่ 1 ตัว
ทุก round นำตัวแรกของ unsorted ไปสอดแทรกในตำแหน่งที่ถูกต้องใน sorted → ไม่ swap แต่ copy (เลื่อน)
จำง่าย: "หยิบ 1 ใบ สอดเข้าที่ถูก" — ต่างจาก Selection ที่ swap จากปลาย

Insertion Sort — Python
def insertion_sort(A):
for i in range(1, len(A)): # เริ่มจาก index 1
key = A[i] # ตัวที่จะแทรก
j = i - 1 # ชี้ตัวสุดท้ายใน sorted
while j >= 0 and A[j] > key:
A[j + 1] = A[j] # เลื่อนขวา (copy ไม่ใช่ swap)
j -= 1
A[j + 1] = key # วาง key ตรงตำแหน่งที่ถูกต้อง
return A
สังเกต: ใช้ copy (A[j+1] = A[j]) ไม่ใช่ swap → ลดการ copy ลงครึ่งนึงเทียบกับ swap
ถ้าข้อมูล "เกือบเรียง" อยู่แล้ว inner while loop จะสั้นมาก → Best Case O(n) ทำงานได้ดีมาก

Complexity Analysis — ประสิทธิภาพของทั้ง 3 Algorithm
ออกสอบแน่ — ตารางนี้ต้องท่องได้
AlgorithmBest CaseAverage CaseWorst CaseSwaps/Copies
Bubble Sort Ω(n) θ(n²) O(n²) Swap ≤ O(n²)
Selection Sort Ω(n²) θ(n²) O(n²) Swap ≤ n-1 ✓
Insertion Sort Ω(n) θ(n²) O(n²) Copy ≤ O(n²)
จุดต่างสำคัญ: Selection Sort Best Case = Ω(n²) — แม้ข้อมูลเรียงแล้ว ยังต้องไล่หา minimum ทุก round
Bubble และ Insertion สามารถหยุดได้เร็วกว่าถ้าข้อมูลเรียงแล้ว → Best Case = Ω(n)
ออกสอบ — สรุป Comparisons และ Swaps สำหรับ n element
Comparisons ทุกตัว
n=1 → 0 ครั้ง
n=2 → 1 ครั้ง
n=3 → 3 ครั้ง
n=6 → 15 ครั้ง
n ตัว → n(n-1)/2 ครั้ง = O(n²)
Swaps (Bubble/Selection)
Bubble: min=0, max=n(n-1)/2
Selection: min=0, max=n-1 ← น้อยมาก!
Insertion: ไม่ swap แต่ copy แทน
min=0, max=n(n-1)/2
Space Complexity ทุกตัว = O(n) — ใช้แค่ array เดิม + ตัวแปรเล็กน้อย

เปรียบเทียบ 3 Algorithm — เลือกใช้อะไรเมื่อไหร่
ออกสอบ — จุดเด่น-ด้อยของแต่ละตัว
🫧 Bubble Sort
✓ เข้าใจง่ายที่สุด
✓ Best case O(n) ถ้าใช้ flag
✗ ช้าที่สุดในทางปฏิบัติ
✗ Swap เยอะมากใน worst case

เหมาะกับ: ข้อมูลน้อย หรือสอนให้เข้าใจ concept
🎯 Selection Sort
✓ Swap น้อยที่สุด (≤ n-1 ครั้ง)
✓ เหมาะเมื่อ cost ของ swap แพง
✗ Best case = Ω(n²) เสมอ
✗ ไม่สามารถ early-stop ได้

เหมาะกับ: เมื่อ memory write มีราคาแพง
🃏 Insertion Sort — ดีที่สุดในบรรดา 3 ตัวนี้
✓ Best case O(n) — เร็วมากถ้าข้อมูลเกือบเรียงแล้ว
✓ เร็วกว่า Bubble Sort ประมาณ 2 เท่าในทางปฏิบัติ
✓ ใช้ copy แทน swap → ลด memory access
✓ เป็น Stable Sort — รักษาลำดับของตัวที่ค่าเท่ากัน
✓ Adaptive — ยิ่งเรียงอยู่แล้วยิ่งเร็ว

เหมาะกับ: ข้อมูลน้อย หรือข้อมูลที่เกือบเรียงอยู่แล้ว — ดีที่สุดในสามตัวนี้เกือบทุกสถานการณ์
Stable Sort คืออะไร — ต้องรู้
Stable Sort = ถ้ามีค่าเท่ากัน 2 ตัว ลำดับสัมพัทธ์ของมันจะไม่เปลี่ยนหลัง sort
Bubble, Selection, Insertion ทั้ง 3 ตัวเป็น Stable Sort ทั้งหมด
เช่น มี [3a, 1, 3b, 2] → หลัง sort ได้ [1, 2, 3a, 3b] — 3a ยังอยู่ก่อน 3b เสมอ
สำคัญมากเมื่อ sort หลายรอบหรือ sort ตาม key หลายตัว
สูตรจำเร็ว ก่อนสอบ
Bubble = เปรียบเทียบคู่ข้างๆ swap ถ้าผิด
→
Selection = หา min swap มาหน้า
→
Insertion = หยิบมา 1 ตัว สอดเข้าที่ถูกต้อง
ทั้ง 3 ตัว: Worst/Average = O(n²), Space = O(n)
Best Case: Bubble = Ω(n), Selection = Ω(n²), Insertion = Ω(n)
