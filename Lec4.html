Graph คืออะไร — ก่อนจะ Traverse ต้องรู้จักก่อน
concept
Graph = โครงสร้างข้อมูลที่ประกอบด้วย Vertex (โหนด/จุด) และ Edge (เส้นเชื่อม)
ต่างจาก Tree ตรงที่ Graph ไม่มี root และ วนลูปได้ (cycle) — โหนดเชื่อมกันได้อิสระกว่า
ใช้แทนความสัมพันธ์จริงในชีวิต เช่น แผนที่, เครือข่าย, โซเชียล, เส้นทางรถไฟ
เปรียบเทียบง่าย ๆ:
Tree = ต้นไม้มีรากชัดเจน เดินได้ทางเดียว
Graph = แผนที่เมือง — ไปจาก A ถึง B ได้หลายเส้นทาง วนกลับได้ด้วย
ออกสอบ — ประเภทของ Graph
Undirected Graph
เส้นเชื่อมไม่มีทิศทาง — A→B และ B→A เหมือนกัน เช่น เพื่อนบน Facebook
Directed Graph (Digraph)
เส้นเชื่อมมีทิศทาง — A→B ≠ B→A เช่น การ Follow บน Twitter/X
Weighted Graph
แต่ละ edge มีน้ำหนัก (cost/distance) เช่น ระยะทางระหว่างเมือง
Connected Graph
ทุก vertex เชื่อมถึงกันได้ทั้งหมด — ไม่มีโหนดโดดเดี่ยว
2
Graph Representation — เก็บ Graph ในโค้ดยังไง
ออกสอบ — 2 วิธีหลัก ต้องเปรียบเทียบได้
1. Adjacency List (dict)
เก็บแต่ละ vertex → list ของ neighbor ที่เชื่อมอยู่

ดีเมื่อ: graph มี edge น้อย (sparse)
ประหยัด memory
หา neighbor ของ node ได้เร็ว
2. Adjacency Matrix (2D array)
เก็บเป็น matrix n×n — ใส่ 1 ถ้าเชื่อม ใส่ 0 ถ้าไม่เชื่อม

ดีเมื่อ: graph หนาแน่น (dense)
เช็คว่า A เชื่อม B ไหมได้ O(1) ทันที
จำง่าย: List = เบา เร็วหา neighbor | Matrix = หนัก แต่เช็ค edge O(1)

สร้าง matrix ขนาด n × n เติม 0 ทั้งหมดก่อน แล้วไล่ใส่ 1 ตาม edge ที่มี

def to_adjacency_matrix(graph):
vertices = sorted(graph.keys()) # เรียงชื่อ vertex
n = len(vertices)
idx = {v: i for i, v in enumerate(vertices)}
matrix = [[0] * n for _ in range(n)] # เริ่มต้น 0 ทั้งหมด
for v, neighbors in graph.items():
for u in neighbors:
matrix[idx[v]][idx[u]] = 1 # ใส่ 1 ตรง edge
return vertices, matrix

Breadth-First Traversal (BFT) — เดินแนวกว้าง ทีละ layer
intuition — เข้าใจก่อนจำ
BFT = เหมือน โยนก้อนหินลงน้ำ — คลื่นกระจายออกทีละวง (layer by layer)
จาก vertex เริ่มต้น → เยี่ยม เพื่อนบ้านทุกคน ก่อน → แล้วค่อยไปไกลขึ้น
ใช้ Queue (FIFO) — ใครเข้าก่อน ออกก่อน เลยได้ผลแบบ level by level
G1 เริ่มจาก A:
Layer 0: A
Layer 1: B, C, E (เพื่อนบ้านตรง ๆ ของ A)
Layer 2: D (เพื่อนของ B และ C ที่ยังไม่เคยเยี่ยม)
ผลลัพธ์: A → B → C → E → D
ออกสอบ — ขั้นตอน BFT จำให้ได้
1. เพิ่ม start ลง Queue
→
2. Mark visited
→
3. Dequeue ออกมา
→
4. เยี่ยม neighbor ที่ยังไม่เยี่ยม → Enqueue
→
5. ทำซ้ำจนหมด
กุญแจสำคัญ: ต้องมี visited set — ป้องกันวนลูปซ้ำ (graph มี cycle ได้!)
BFT — Python implementation
from collections import deque

def breadth_first_traversal(graph, start):
visited = set() # เก็บโหนดที่เยี่ยมแล้ว
queue = deque([start]) # เริ่มต้นด้วย start node
visited.add(start)

while queue:
vertex = queue.popleft() # ดึงออกจากหน้า queue
print(vertex, end=' → ')

for neighbor in graph[vertex]:
if neighbor not in visited:
visited.add(neighbor)
queue.append(neighbor) # เพิ่มต่อหลัง queue
สังเกต: visited.add() ก่อน append เข้า queue เลย
→ ป้องกันโหนดเดิมถูก enqueue ซ้ำหลายครั้ง ซึ่งจะทำให้ผลผิด

Depth-First Traversal (DFT) — เดินแนวลึก ดำดิ่งก่อน
intuition — เข้าใจก่อนจำ
DFT = เหมือน สำรวจถ้ำ — เดินลึกสุดทางเดียวก่อน ค่อยถอยมาลองทางอื่น
จาก vertex เริ่มต้น → ดำลึกไปเรื่อย ๆ จน dead-end → backtrack → ลองทางใหม่
ใช้ Recursion (หรือ Stack) — call stack คือ memory ที่จำว่าต้องถอยไปไหน
G1 เริ่มจาก A (สมมติไล่ตาม alphabet):
A → B → D → C → E
(ดำลึกก่อน ไม่ออกกว้าง — ต่างจาก BFT ที่ได้ A → B → C → E → D)
ออกสอบ — ขั้นตอน DFT จำให้ได้
1. Mark visited
→
2. เยี่ยม vertex นี้
→
3. loop neighbor ที่ยังไม่เยี่ยม
→
4. เรียก DFT(neighbor) ซ้ำ (recursive)
เหมือน BFT แต่เปลี่ยน Queue → Recursion และ ดำลึกก่อน แทนที่จะกระจายกว้าง
DFT — Python implementation (recursive)
def depth_first_traversal(graph, vertex, visited=None):
if visited is None:
visited = set() # สร้าง visited ครั้งแรก

visited.add(vertex) # mark ก่อนเยี่ยม
print(vertex, end=' → ')

for neighbor in graph[vertex]:
if neighbor not in visited:
depth_first_traversal(graph, neighbor, visited) # ดำลึก!
จุดสำคัญ: visited ส่งต่อเป็น argument เดิมทุก recursive call
→ ทุก level รู้ว่าเยี่ยมอะไรไปแล้ว ไม่วนซ้ำ

BFT vs DFT — เปรียบเทียบให้ชัด (Lab 4)
ออกสอบ — ต้องบอกความแตกต่างได้
หัวข้อBFT (แนวกว้าง)DFT (แนวลึก)
โครงสร้างที่ใช้Queue (deque)Recursion / Stack
ลำดับการเยี่ยมLayer by layerดำลึกก่อน backtrack
เหมาะกับหาเส้นทางสั้นสุด (Shortest Path)ตรวจ cycle, topological sort
Memoryใช้มากถ้า graph กว้างใช้มากถ้า graph ลึก (stack overflow)
ผลลัพธ์คล้ายกัน แต่ลำดับต่างกันคล้ายกัน แต่ลำดับต่างกัน
ทั้งสองเยี่ยมทุก vertex เหมือนกัน (ถ้า graph connected) — แต่ ลำดับการเยี่ยมต่างกัน
เปรียบเทียบง่าย: BFT = น้ำท่วมไล่ระดับ | DFT = งูเลื้อยลึก
สิ่งที่เหมือนกัน ทั้ง BFT และ DFT
ทั้งคู่ต้องมี visited set — ป้องกัน infinite loop ใน graph ที่มี cycle
ทั้งคู่รับ graph (adjacency list) และ start vertex เป็น input
ทั้งคู่ เยี่ยมทุก vertex ครั้งเดียว — complexity เท่ากันคือ O(V + E)
O(V + E) หมายถึง: V = จำนวน vertex, E = จำนวน edge
ไล่ทุก node หนึ่งครั้ง + ดู edge ทุกเส้นหนึ่งครั้ง — ไม่มีการทำซ้ำ
