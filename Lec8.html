Selection Problem (ปัญหาการเลือก) คืออะไร?
concept
Selection คือการหาข้อมูล อันดับที่ k จากข้อมูลที่ยังไม่ได้เรียงลำดับ (Unsorted)
เช่น ต้องการรู้ว่า "คะแนนสอบสูงสุดอันดับ 3" หรือ "ราคา median" คือเท่าใด โดยไม่ต้องเรียงลำดับข้อมูลทั้งหมดก่อน
k=1 คือค่าน้อยสุด, k=n คือค่ามากสุด, k=⌊n/2⌋ คือ median
อุปมา: สมมติมีนักเรียน 30 คนถือตั๋วหมายเลขสุ่มไว้ เราต้องการรู้ว่าคนถือหมายเลขอันดับที่ 10 คือใคร
วิธีง่าย: เรียงทุกคน → ดูคนที่ 10 — แต่ใช้เวลามาก!
วิธีฉลาด: แบ่งกลุ่มด้วยตัวกลาง → ตัดกลุ่มที่ไม่ใช่คำตอบออก → ทำซ้ำในกลุ่มที่เหลือ
เปรียบประสิทธิภาพ
❌ วิธีซื่อ — Sort แล้วเข้าถึง
Sort ข้อมูลทั้งหมดก่อน แล้วจึงดูค่าที่ตำแหน่ง k

Time: O(n log n)
ทำงานมากเกินความจำเป็น เพราะ Sort ข้อมูลทั้งหมดทั้งที่ต้องการแค่ตัวเดียว
✅ วิธีฉลาด — Quick Select
ใช้เทคนิค Prune-and-Search ตัดข้อมูลที่ไม่ต้องการออกทีละรอบ

Time: O(n) expected
เร็วกว่ามาก โดยไม่จำเป็นต้อง sort ทั้งหมดเลย
2
Prune-and-Search — เทคนิคสำคัญ
concept
Prune-and-Search (หรือ Decrease-and-Conquer) คือเทคนิคที่ ตัด (Prune) ข้อมูลส่วนที่แน่ใจว่าไม่ใช่คำตอบออก แล้วเรียกซ้ำ (Recursive) ในส่วนที่เหลือ
เทคนิคนี้เหมือนกับที่ใช้ใน Binary Search แต่ใช้กับข้อมูลที่ยังไม่เรียงลำดับ
Binary Search ต้องการข้อมูลเรียงแล้ว → ตัดครึ่งทีละรอบ
Prune-and-Search ไม่ต้องการข้อมูลเรียงแล้ว → ตัดส่วนที่ไม่ต้องการออก แต่ไม่จำเป็นต้องตัดครึ่งเสมอ
2 ขั้นตอนหลักที่ต้องจำ
✂️ Prune (ขจัด)
ตัดข้อมูลบางส่วนที่แน่ใจว่าไม่ใช่คำตอบออก แล้วเรียกซ้ำ (Recursive) หาค่าอันดับที่ k จากข้อมูลที่เหลือ
🔍 Search (ค้นหา)
เมื่อข้อมูลที่เหลือมีจำนวนเท่ากับค่าคงที่ที่กำหนด ให้ค้นหาด้วย Brute-Force แล้วคืนค่ากลับขึ้นไป

Randomized Quick Select — วิธีทำทีละขั้น
intuition
เริ่มต้นด้วยการ สุ่มเลือก Pivot 1 ตัวจากข้อมูล
เปรียบเทียบ Pivot กับข้อมูลทุกตัว แล้ว แบ่งออกเป็น 3 กลุ่ม:
A1 — น้อยกว่า Pivot
ข้อมูลทุกตัวที่มีค่า < Pivot รวมกันเป็นกลุ่ม A1
A2 — เท่ากับ Pivot
ข้อมูลทุกตัวที่มีค่า = Pivot รวมกัน (รวม Pivot ตัวเองด้วย)
A3 — มากกว่า Pivot
ข้อมูลทุกตัวที่มีค่า > Pivot รวมกันเป็นกลุ่ม A3
จากนั้น เปรียบ k กับขนาดแต่ละกลุ่ม เพื่อตัดสินว่าจะค้นต่อในกลุ่มไหน:
# กฎการตัดสิน (Decision Rule) if k <= |A1|: # คำตอบอยู่ใน A1 quick_select(A1, k) elif k <= |A1| + |A2|: # คำตอบคือ Pivot เอง! return pivot else: # คำตอบอยู่ใน A3 new_k = k - |A1| - |A2| quick_select(A3, new_k)
ตัวอย่างจากสไลด์ — Walk-through ทีละรอบ
ข้อมูล A = [55, 29, 32, 43, 43, 9, 19, 90, 76, 13, 43, 29, 65, 88, 76], หา k=6
รอบที่ 1: Pivot = 43
  A1 (น้อยกว่า 43) = [29,32,9,19,13,29] → |A1|=6
  A2 (เท่ากับ 43) = [43,43,43] → |A2|=3
  A3 (มากกว่า 43) = [55,90,76,65,88,76] → |A3|=6
ตรวจสอบ: k=6 ≤ |A1|=6 → เรียกซ้ำกับ A1, k=6
รอบที่ 2: ข้อมูลคือ A1=[29,32,9,19,13,29], Pivot = 29
  A11 = [9,19,13] → |A11|=3
  A12 = [29,29] → |A12|=2
  A13 = [32] → |A13|=1
ตรวจสอบ: k=6 > |A11|+|A12|=5 → k ใหม่ = 6−3−2 = 1
เรียกซ้ำกับ A13=[32], k=1
รอบที่ 3: ข้อมูลคือ [32], k=1, |A13|=1
→ มีข้อมูลตัวเดียวและ k=1 → คืนค่า 32 ✓

ตรวจสอบ: เรียงจริงคือ [9,13,19,29,29,32,43,43,43,55,...] → อันดับที่ 6 = 32 ✅

Time Complexity — ทำไม O(n)?
วิเคราะห์ความซับซ้อน
Expected Time: O(n) ภายใต้สมมติฐานว่าการเปรียบเทียบแต่ละคู่ใช้เวลา O(1)
แต่ละรอบทำงาน O(n) เพื่อแบ่ง 3 กลุ่ม แต่รอบถัดไปข้อมูลจะลดลงเรื่อยๆ
ถ้า Pivot ดี (แบ่งข้อมูลได้ประมาณครึ่ง) งานแต่ละรอบลดลงแบบ:
n + n/2 + n/4 + n/8 + ... ≈ 2n = O(n)
กรณีComplexityเหตุผล
Best Case (Pivot ดีมาก) Ω(n) ตัดข้อมูลได้ครึ่งทุกรอบ รวมแค่ 2n
Expected (เฉลี่ย) O(n) โดยเฉลี่ยแล้ว Pivot ที่สุ่มได้จะดีพอ
Worst Case (Pivot แย่มาก) O(n²) ได้ min หรือ max ทุกรอบ ตัดได้แค่ 1 ตัว
Sort แล้วเข้าถึง (เปรียบ) O(n log n) Merge/Quick Sort ช้ากว่า Quick Select
ทำไมต้องสุ่ม Pivot?
การสุ่มทำให้โอกาสได้ Pivot แย่ทุกรอบน้อยมาก ทำให้ Expected Time เป็น O(n) แม้ worst case ยังเป็น O(n²)
Median of Medians
วิธีขั้นสูงกว่าที่รับประกัน O(n) แม้ worst case โดยการเลือก Pivot อย่างชาญฉลาด (ศึกษาเพิ่มใน Textbook)

สรุปย่อ — จำก่อนสอบ
cheatsheet
📌 Selection Problem
หาค่าอันดับที่ k จาก array ไม่เรียงลำดับ ใน O(n) expected
✂️ Prune-and-Search
ตัดข้อมูลที่ไม่ต้องการออกทีละรอบ แล้วเรียกซ้ำในส่วนที่เหลือ
🎲 Pivot + 3 Groups
สุ่ม Pivot → แบ่ง A1(<), A2(=), A3(>) → ตัด 2 กลุ่มออก
⚡ Complexity
Expected O(n) · Worst O(n²) · Better than Sort O(n log n)
กฎตัดสิน (จำให้ขึ้นใจ!):
k ≤ |A1| → ค้นใน A1, k เดิม
k ≤ |A1|+|A2| → คืน Pivot ทันที!
k > |A1|+|A2| → ค้นใน A3, k ใหม่ = k − |A1| − |A2|
