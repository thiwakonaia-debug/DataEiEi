แนวคิด Divide and Conquer (แบ่งแยกและเอาชนะ)
concept
Divide and Conquer คือเทคนิคการแก้ปัญหาแบบ Recursive โดยการแบ่งปัญหาใหญ่ให้เล็กลงเรื่อยๆ
ประกอบด้วย 3 ขั้นตอนหลัก: Divide (แบ่ง) → Conquer (เอาชนะ/แก้) → Combine (รวม)
อุปมา: เหมือนการแบ่งงานให้เพื่อนในกลุ่มทำคนละส่วน แล้วสุดท้ายค่อยเอางานของทุกคนมารวมเป็นเล่มเดียว
3 ขั้นตอนที่ต้องจำ
1. Divide
แบ่งปัญหาออกเป็นปัญหาย่อยๆ (Sub-problems) ที่มีลักษณะเหมือนเดิมแต่ขนาดเล็กลง
2. Conquer
แก้ปัญหาย่อยด้วย Recursion ถ้าปัญหาเล็กพอ (Base Case) ให้แก้โดยตรงทันที
3. Combine
นำคำตอบจากปัญหาย่อยๆ มารวมกันเพื่อให้ได้คำตอบของปัญหาดั้งเดิม
2
Merge Sort — เรียงลำดับด้วยการแบ่งครึ่งและรวม
intuition
Merge Sort จะแบ่ง Array ออกเป็น 2 ส่วนเท่าๆ กันเสมอ จนเหลือตัวเดียว (ซึ่งถือว่าเรียงแล้ว) แล้วค่อย Merge (รวม) กลับขึ้นมาโดยเรียงลำดับไปด้วย
จุดเด่น: Predictable Performance — ทำงาน O(n log n) เสมอ ไม่ว่าข้อมูลจะมาแบบไหน
จุดด้อย: ใช้ Memory เยอะ — ต้องสร้าง Array สำรอง (Auxiliary Array) ขนาด O(n)

Complexity Analysis
Time: O(n log n)
การแบ่งครึ่งทำให้ต้นไม้สูง log n ชั้น แต่ละชั้นมีการ Merge ทำงาน O(n)
→ ทุกกรณี (Best/Avg/Worst) คือ O(n log n)
Space: O(n)
ต้องใช้พื้นที่เก็บข้อมูลชั่วคราวระหว่างการ Merge เท่ากับจำนวนข้อมูลดั้งเดิม
Merge Sort เป็น Stable Sort (รักษาลำดับของค่าที่เท่ากัน)
3
Quick Sort — เรียงลำดับด้วยการเลือก Pivot
intuition
Quick Sort เลือกค่าหนึ่งมาเป็น Pivot แล้วจัดระเบียบ Array ให้:
[ตัวที่เล็กกว่า pivot] | Pivot | [ตัวที่ใหญ่กว่า pivot]
ทำแบบเดิมซ้ำๆ (Recursion) กับฝั่งซ้ายและฝั่งขวา
จุดเด่น: In-place Sort — ไม่ต้องใช้ Memory สำรองเยอะ (ใช้แค่ Call Stack) และมักจะเร็วที่สุดในทางปฏิบัติ

Complexity Analysis
Avg Case: O(n log n)
ถ้าเลือก pivot ได้ดี (แบ่งสองฝั่งได้พอๆ กัน) จะเร็วมาก และใช้พื้นที่น้อย O(log n)
Worst Case: O(n²)
ถ้าเลือก pivot ได้แย่ที่สุด (ได้ตัว min หรือ max ทุกรอบ) เช่น array เรียงอยู่แล้วแต่เลือก pivot ตัวแรก
Quick Sort ไม่ใช่ Stable Sort (ลำดับค่าที่เท่ากันอาจเปลี่ยนไป)

AlgorithmBest CaseAvg/WorstSpaceStable
