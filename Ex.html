Tree Traversal คืออะไร — ก่อนเริ่ม
concept
Tree Traversal = การเยี่ยม (visit) ทุก node ใน tree อย่างเป็นระบบ ไม่เยี่ยมซ้ำ ไม่ข้ามใครไว้
ลำดับที่เลือกเยี่ยมต่างกัน → ได้ผลลัพธ์ต่างกัน → ใช้งานต่างกัน
มี 2 กลุ่มหลัก: DFS (ดำดิ่งลึกก่อน) และ BFS (กว้างออก ทีละ level)
ต้นไม้ตัวอย่างที่ใช้ตลอดเอกสารนี้:

         A       ← root
       /   \
     B     C     ← internal nodes
    / \     \
   D   E     F   ← leaves
1
Preorder — Root → Left → Right
แนวคิดหลัก
เยี่ยมตัวเองก่อน แล้วค่อยลงซ้าย แล้วลงขวา
จำสั้น ๆ: NLR (Node → Left → Right) หรือ "Root ก่อนเสมอ"
เหมือนอ่านหนังสือ — เห็นชื่อบทก่อน แล้วค่อยอ่านเนื้อหา
ลำดับการเดิน — กด Play ดูได้เลย
Python Code
def preorder(node): if node is None: return visit(node) # 1. เยี่ยมตัวเอง (Root) ก่อน preorder(node.left) # 2. ลงซ้าย preorder(node.right) # 3. ลงขวา # Output: A → B → D → E → C → F
เหมาะใช้เมื่อ
Copy หรือ Clone tree — ต้องสร้าง parent ก่อน ค่อยสร้างลูก
Serialize/Save tree — บันทึกโครงสร้าง tree ลง file ให้โหลดกลับมาได้
Print directory structure — เหมือน ls -R ใน terminal
2
Inorder — Left → Root → Right
แนวคิดหลัก
ลงซ้ายจนสุด แล้วค่อยเยี่ยมตัวเอง แล้วไปขวา
จำสั้น ๆ: LNR (Left → Node → Right) หรือ "Root อยู่กลาง"
🌟 Superpower: ใน Binary Search Tree (BST) จะได้ข้อมูล เรียงจากน้อยไปมาก โดยอัตโนมัติ!
ลำดับการเดิน — กด Play ดูได้เลย
Python Code
def inorder(node): if node is None: return inorder(node.left) # 1. ลงซ้ายก่อน visit(node) # 2. เยี่ยมตัวเอง (Root) inorder(node.right) # 3. ลงขวา # Output: D → B → E → A → C → F # ใน BST จะได้ค่าเรียงเสมอ! ✨
เหมาะใช้เมื่อ
BST → Sorted list — ได้ข้อมูลเรียงโดยไม่ต้อง sort เพิ่ม
Expression tree — ได้ infix notation เช่น 3+5*2
Validate BST — ตรวจว่า tree เป็น BST ถูกต้องไหม
3
Postorder — Left → Right → Root
แนวคิดหลัก
ลูกต้องถูกเยี่ยมทั้งหมดก่อน แล้ว Root จึงถูกเยี่ยมสุดท้าย
จำสั้น ๆ: LRN (Left → Right → Node) หรือ "Root สุดท้ายเสมอ"
เหมือน "ทำการบ้านให้เสร็จก่อน แล้วค่อยส่ง" — เสร็จจากล่างขึ้นบน
ลำดับการเดิน — กด Play ดูได้เลย
Python Code
def postorder(node): if node is None: return postorder(node.left) # 1. ลงซ้ายก่อน postorder(node.right) # 2. ลงขวา visit(node) # 3. เยี่ยมตัวเอง (Root) สุดท้าย # Output: D → E → B → F → C → A # สังเกต: Root (A) อยู่สุดท้ายเสมอ!
เหมาะใช้เมื่อ
Delete tree — ลบลูกก่อน แล้วค่อยลบ parent (ไม่งั้น memory leak)
Expression calculator — คำนวณ postfix: 3 5 + 2 * = 16
Directory size — นับขนาดไฟล์จากล่างขึ้นบน (ลูกก่อน แล้วรวมขึ้นพ่อ)
4
BFS — Breadth-First Search (Level-order)
แนวคิดหลัก
เยี่ยมทีละ Level จากบนลงล่าง ซ้ายไปขวา — ไม่ดำดิ่งลึกก่อน
ใช้ Queue (FIFO) — ใครมาก่อนออกก่อน เหมือนคิวธนาคาร
🎯 รับประกัน: node ใกล้ root จะถูกเยี่ยมก่อนเสมอ → หาเส้นทางสั้นสุดได้
ลำดับการเดิน — กด Play ดูได้เลย
Python Code — ใช้ Queue
from collections import deque def bfs(root): if root is None: return queue = deque([root]) # ใส่ root ลง Queue while queue: node = queue.popleft() # นำออกจากหน้า (FIFO) visit(node) if node.left: queue.append(node.left) if node.right: queue.append(node.right) # Output: A → B → C → D → E → F
เหมาะใช้เมื่อ
Shortest path — GPS, Maps หาเส้นทางสั้นสุด
Social network — หา degree of separation ระหว่างคน
Level-order print — พิมพ์ tree ทีละชั้น
5
DFS — Depth-First Search (Generic)
แนวคิดหลัก
ดำดิ่งลึกในกิ่งหนึ่งจนสุด ก่อนย้อนกลับมาลองกิ่งถัดไป
ใช้ Stack (LIFO) หรือ Recursion — ใส่ขวาก่อน ซ้ายทีหลัง (ซ้ายออกก่อน)
⚠️ Preorder / Inorder / Postorder ทั้ง 3 แบบ ล้วนเป็น DFS ต่างกันแค่จังหวะเยี่ยม Root
ลำดับการเดิน — กด Play ดูได้เลย
Python Code — ใช้ Stack (Iterative)
def dfs(root): if root is None: return stack = [root] # ใช้ Stack แทน Queue while stack: node = stack.pop() # นำออกจากบน (LIFO) visit(node) if node.right: stack.append(node.right) # ใส่ขวาก่อน if node.left: stack.append(node.left) # ใส่ซ้ายทีหลัง (ออกก่อน) # Output: A → B → D → E → C → F (เหมือน Preorder)
เหมาะใช้เมื่อ
Pathfinding — ค้นหาว่ามีเส้นทางจาก A → B ไหม
Cycle detection — ตรวจว่า graph มี cycle ไหม
Topological sort — จัดลำดับ dependency ของ tasks
6
BFS vs DFS — เปรียบเทียบชัดเจน
🌊 BFS — คลื่นน้ำ
• ใช้ Queue (FIFO)
• เยี่ยมทีละ level กว้างออก
• memory สูง (เก็บทั้ง level)
• รับประกัน Shortest Path ✅
• เหมาะกับ tree ตื้น กว้าง
🕳️ DFS — นักสำรวจถ้ำ
• ใช้ Stack / Recursion
• ดำดิ่งลึกในกิ่งก่อน
• memory ต่ำกว่า (เก็บแค่ depth)
• ไม่รับประกัน Shortest Path ❌
• เหมาะกับ tree ลึก แคบ
อุปมา:
BFS = หยดน้ำบนพื้น → กระจายออกกว้าง ทีละชั้นเท่า ๆ กัน
DFS = นักสำรวจถ้ำ → เลือกทางแล้วเดินลึกสุด แล้วค่อยย้อนกลับมาลองทางอื่น
7
ตารางสรุปเปรียบเทียบทุก Traversal
Traversal สูตร ผลลัพธ์ (A,B,C,D,E,F) Data Structure Trick จำ ใช้เมื่อ
Preorder NLR A B D E C F Stack / Recursion Root ก่อนเสมอ Copy, Serialize
Inorder LNR D B E A C F Stack / Recursion Root อยู่กลาง BST Sort, Validate
Postorder LRN D E B F C A Stack / Recursion Root สุดท้ายเสมอ Delete, Calculator
BFS Level-by-Level A B C D E F Queue (FIFO) คลื่นน้ำ ทีละชั้น Shortest Path
DFS Dive Deep First A B D E C F Stack (LIFO) นักสำรวจถ้ำ Path, Cycle, Topo
Cheat Sheet — จำก่อนสอบ:
Pre/In/Post → ดูตำแหน่ง N (Node/Root): NLR · LNR · LRN
BFS → Queue · DFS → Stack
Inorder + BST = เรียงอัตโนมัติ 🎯 · Postorder Root = สุดท้ายเสมอ 🎯
