Binary Tree — พื้นฐานก่อนเข้า BST
concept
Binary Tree = tree ที่แต่ละ node มีลูกได้ ไม่เกิน 2 ตัว คือ left child และ right child
แต่ละ node เก็บ: ข้อมูล (value), left subtree, right subtree
ต่างจาก general tree ตรงที่ restrict จำนวนลูก — ทำให้ traverse และ search ได้ง่ายกว่า
ออกสอบ — ประเภทของ Binary Tree
Full Binary Tree
ทุก node มีลูก 0 หรือ 2 ตัวเท่านั้น — ห้ามมีลูกแค่ 1 ตัว
Complete Binary Tree
ทุก level เต็มหมด ยกเว้น level สุดท้ายที่ไล่จากซ้ายไปขวา
Balanced Binary Tree
ความสูงของ left กับ right subtree ต่างกันได้ไม่เกิน 1
Perfect Binary Tree
ทุก internal node มีลูก 2 คน และทุก leaf อยู่ที่ depth เดียวกันเป๊ะ
จำง่าย: Full = 0 หรือ 2, Complete = เติมซ้ายก่อน, Balanced = สมดุล, Perfect = สมบูรณ์แบบที่สุด
เปรียบเทียบ
FeatureBinary TreeGeneral Tree
ลูกสูงสุด2 ตัวไม่จำกัด
โครงสร้างStrict hierarchyFlexible hierarchy
ประสิทธิภาพเร็วกว่าช้ากว่า
ใช้ในBST, Heap, Expression TreeFile system, Decision tree

LinkedBinaryTree — ขยายจาก LinkedTree
concept
LinkedBinaryTree extends LinkedTree — ไม่ต้องเขียนใหม่ทั้งหมด แค่ override method ที่จำเป็น
เพิ่ม 2 method หลัก: add_left(p, e) และ add_right(p, e) เพื่อ restrict ลูกให้ไม่เกิน 2
เพิ่ม accessor: left(p) คืน left child, right(p) คืน right child
implementation
class LinkedBinaryTree(LinkedTree):
def add_left(self, p, e):
node = self._validate(p)
if len(node._children) > 0:
raise ValueError("Left child already exists!")
return self.add_child(p, e)

def add_right(self, p, e):
node = self._validate(p)
if len(node._children) > 1:
raise ValueError("Right child already exists!")
return self.add_child(p, e)

def left(self, p): # ได้ left child หรือ None
return next(iter(self.children(p)), None)

def right(self, p): # ได้ right child หรือ None
children = list(self.children(p))
return children[1] if len(children) > 1 else None
ตัวอย่างสร้าง tree: bt.add_child(None, "A") → root A
bt.add_left(root, "B") → B เป็น left child
bt.add_right(root, "C") → C เป็น right child

Tree Traversal — 4 วิธีเดิน tree
ออกสอบบ่อยมาก — จำลำดับให้ได้
Traversalลำดับใช้ทำอะไร
PreorderRoot → Left → RightCopy tree, Expression tree
InorderLeft → Root → RightBST sorted order ✓
PostorderLeft → Right → Rootลบ tree, คำนวณ expression
BFS (Level-order)Level by levelShortest path, AI decision
จำง่าย — ชื่อบอกเลยว่า "root" อยู่ที่ไหน:
PRE-order = root ก่อน (pre = ก่อน)
IN-order = root กลาง (in = กลาง)
POST-order = root หลัง (post = หลัง)
โค้ด 3 traversal
# Preorder: Root → Left → Right
def preorder_traversal(self, p):
if p:
print(p.element(), end=' → ')
self.preorder_traversal(self.left(p))
self.preorder_traversal(self.right(p))

# Inorder: Left → Root → Right
def inorder_traversal(self, p):
if p:
self.inorder_traversal(self.left(p))
print(p.element(), end=' → ')
self.inorder_traversal(self.right(p))

# Postorder: Left → Right → Root
def postorder_traversal(self, p):
if p:
self.postorder_traversal(self.left(p))
self.postorder_traversal(self.right(p))
print(p.element(), end=' → ')
BFS ใช้ Queue
def breadth_first_traversal(self):
if self.is_empty(): return
queue = deque([self.root()]) # ใช้ deque เป็น queue
while queue:
p = queue.popleft() # ดึงออกจากหน้า
print(p.element(), end=' → ')
for child in self.children(p):
queue.append(child) # เพิ่มลูกเข้าหลัง queue
BFS ใช้ Queue (FIFO) — ทำงานทีละ level จากบนลงล่าง ต่างจาก DFS ที่ดำดิ่งลึกก่อน

การลบ Node — 3 กรณี (ออกสอบแน่)
ออกสอบ — 3 cases ต้องจำให้ได้ทุกข้อ
Case 1: Leaf (ไม่มีลูก)
ลบออกตรง ๆ เลย — ง่ายที่สุด แค่เอาออกจาก parent._children
Case 2: มีลูก 1 คน
แทนที่ node ด้วยลูกของมัน — ต่อสายตรงจาก grandparent ไปหาลูก
Case 3: มีลูก 2 คน (ยากที่สุด)
หา inorder successor (node ที่เล็กที่สุดใน right subtree) → คัดลอกค่ามาใส่ → ลบ successor ออก
ออกสอบ — Inorder Successor คืออะไร
Inorder Successor = node ที่มีค่าน้อยที่สุดใน right subtree = leftmost node ของ right subtree
วิธีหา: ไป right child ก่อน แล้วเดินซ้ายไปเรื่อย ๆ จนถึงจุดที่ไม่มี left child
def _find_min(self, p):
current = p
while self.left(current): # เดินซ้ายไปเรื่อย ๆ
current = self.left(current)
return current # leftmost = smallest
ทำไมต้องใช้ inorder successor? เพราะมันเป็นค่าที่เล็กที่สุดที่ยังคงมากกว่า node ที่ถูกลบ — ทำให้ BST property ยังคงอยู่ครบ

Binary Search Tree (BST) — หัวใจของ lecture นี้
ออกสอบ — BST property ต้องท่องได้
BST = Binary Tree ที่มีกฎเพิ่ม: ค่าซ้าย < root < ค่าขวา (ทุก node ในทุก subtree)
Left subtree = เฉพาะค่าที่ น้อยกว่า root
Right subtree = เฉพาะค่าที่ มากกว่า root
Inorder traversal ของ BST → ได้ค่า เรียงจากน้อยไปมากเสมอ
ตัวอย่าง: ใส่ 50, 30, 70, 20, 40, 60, 80
Root = 50 → ซ้าย: 30 (ซ้าย: 20, ขวา: 40) → ขวา: 70 (ซ้าย: 60, ขวา: 80)
Inorder = 20 → 30 → 40 → 50 → 60 → 70 → 80 ✓ เรียงแล้ว!
ออกสอบ — Complexity
Average Case: O(log n)
ต้นไม้สมดุล — ทุกครั้งที่เปรียบเทียบ ตัดทางเลือกออกครึ่งนึง
Worst Case: O(n)
ใส่ข้อมูลเรียงลำดับ → tree กลายเป็นเส้นตรง (skewed tree)
Worst case เกิดเมื่อ insert 1, 2, 3, 4, 5... ตามลำดับ — tree จะโน้มเอียงขวาทั้งหมด กลายเป็นเหมือน linked list
ทำไมต้องใช้ BST
Search O(log n) — เร็วกว่า linear search O(n) มาก
ดึงข้อมูลเรียง (sorted) ได้ทันทีด้วย inorder traversal
รองรับ dynamic data — insert/delete ได้โดยไม่ต้อง rebuild ทั้งหมด
ใช้ใน: Database indexing, Symbol table ใน compiler, AI decision tree

BST Implementation — โค้ดสำคัญ
Insert — recursive
เริ่มจาก root → ถ้าค่าน้อยกว่า ไปซ้าย → ถ้ามากกว่า ไปขวา → ใส่ตรงที่ว่าง
def insert(self, p, e):
if p is None: # tree ว่าง → สร้าง root
return self.add_child(None, e)
node = self._validate(p)
if e < node._element: # น้อยกว่า → ไปซ้าย
if not self.left(p):
return self.add_left(p, e)
else:
return self.insert(self.left(p), e)
else: # มากกว่า → ไปขวา
if not self.right(p):
return self.add_right(p, e)
else:
return self.insert(self.right(p), e)
Search — recursive
ใช้ BST property ตัดสินทุกขั้น — ไม่ต้องดูทุก node แค่เลือกซ้ายหรือขวา
def search(self, p, e):
if p is None: return None # ไม่เจอ
node = self._validate(p)
if e == node._element: return p # เจอแล้ว!
elif e < node._element:
return self.search(self.left(p), e) # ไปซ้าย
else:
return self.search(self.right(p), e) # ไปขวา
