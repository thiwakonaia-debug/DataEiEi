Tree คืออะไร — พื้นฐานสุด
concept
Tree = โครงสร้างข้อมูลแบบลำดับชั้น — ต่างจาก array/linked list ที่เรียงตรง ๆ tree แตกกิ่งออกเหมือนต้นไม้จริง ๆ
ทุก node เชื่อมกันด้วย edge (เส้นเชื่อม) และมีความสัมพันธ์แบบ parent-child
ข้อดีคือ search/retrieve ข้อมูลได้เร็วกว่า linear structure มาก
จินตนาการว่าแผนผังองค์กร — CEO อยู่บนสุด (root), แต่ละแผนกเป็น internal node, พนักงานปลายสายเป็น leaf — นี่คือ tree
ออกสอบบ่อย
คำศัพท์สำคัญ — ต้องจำให้ขึ้นใจ
Root
โหนดบนสุด ไม่มี parent เลย มีได้แค่ 1 ตัวใน tree
Leaf / External node
โหนดที่ไม่มีลูกเลย อยู่ปลายสุดของกิ่ง
Internal node
โหนดที่มีลูกอย่างน้อย 1 คน ไม่ใช่ leaf
Siblings
โหนดที่มี parent เดียวกัน = พี่น้องกัน
Ancestor
บรรพบุรุษ — ทุกโหนดที่อยู่เหนือขึ้นไปจนถึง root รวม node ตัวเองด้วย
Descendant
ลูกหลาน — ทุกโหนดที่อยู่ใต้ node นั้นลงไปทุกระดับ
ตัวอย่าง file system: /user/rt/courses/ คือ root → cs016/ เป็น internal node → hw1, hw2, hw3 เป็น leaf
ตัวอย่าง real-world
Python Exception hierarchy ก็คือ tree — BaseException เป็น root, Exception เป็น internal, KeyError/ValueError เป็น leaf
Ordered tree = ลำดับของลูกสำคัญ เช่น หนังสือ — บท 1 ต้องมาก่อนบท 2 เสมอ
Unordered tree = ลำดับไม่สำคัญ เช่น org chart — ไม่สนว่าแผนกไหนอยู่ซ้ายหรือขวา

Depth และ Height — จุดที่สับสนที่สุด
ออกสอบบ่อยมาก
Depth(p) = นับขั้นจาก root ลงมาถึง p — root มี depth = 0 เสมอ
Height(p) = นับขั้นจาก p ลงไปถึง leaf ที่ลึกที่สุด — leaf มี height = 0 เสมอ
Height ของทั้งต้นไม้ = height ของ root node
จำง่าย: Depth วัด "ลงดิน" จาก root, Height วัด "ขึ้นฟ้า" จาก leaf

ถ้า tree: Electronics R'Us → Sales → International → Overseas → Africa
— depth(Africa) = 4 (นับจาก root ลงมา 4 ขั้น)
— height(Africa) = 0 (เป็น leaf ไม่มีลูก)
— height(Overseas) = 1 (ลึกสุดใต้มันคือ Africa, 1 ขั้น)
— height ของ tree นี้ = 4
ออกสอบ — สูตร recursive
สูตรคำนวณ depth:
def depth(self, p):
if self.is_root(p):
return 0 # base case: root = 0
else:
return 1 + self.depth(self.parent(p)) # ขึ้นหา parent แล้วบวก 1
สูตรคำนวณ height (efficient):
def _height2(self, p):
if self.is_leaf(p):
return 0 # base case: leaf = 0
else:
return 1 + max(self._height2(c) for c in self.children(p))
ออกสอบ — complexity
_height1 (แบบไม่ดี) O(n²)
คำนวณ depth ของทุก leaf แล้วเอามา max — แต่ละ leaf ต้องวิ่งขึ้น root ซ้ำซ้อน
_height2 (แบบดี) O(n)
คำนวณ bottom-up ผ่านแต่ละ node แค่ครั้งเดียว เก็บผลแต่ละ subtree ไว้เลย

Tree ADT — interface มาตรฐาน
concept
ADT = Abstract Data Type — แค่ "สัญญา" ว่าต้องมี method อะไรบ้าง ยังไม่บอกว่า implement ยังไง
เหมือนสัญญาจ้างงาน: บอกว่า "ต้องทำอะไรได้บ้าง" แต่ไม่บอกว่า "ทำยังไง"
T.root()
คืนค่า position ของ root
T.parent(p)
คืนค่า parent ของ p, None ถ้าเป็น root
T.children(p)
iterate ลูกทั้งหมดของ p
T.num_children(p)
จำนวนลูกของ p
T.is_leaf(p)
True ถ้า p ไม่มีลูก
T.is_empty()
True ถ้า tree ว่างเปล่า
abstract vs concrete
Abstract method = raise NotImplementedError ไว้ subclass ต้อง implement ไม่งั้น error
Concrete method = implement แล้วใน base class ใช้ได้เลย เช่น is_root(), is_leaf(), is_empty()
def is_root(self, p):
return self.root() == p # concrete — implement แล้ว

def root(self):
raise NotImplementedError(...) # abstract — subclass ต้อง implement

LinkedTree — การ implement จริง
implementation
แต่ละ node เก็บ 3 อย่าง: _element (ข้อมูล), _parent (pointer ไปหา parent), _children (list ของลูก)
class Node:
__slots__ = '_element', '_parent', '_children'
def __init__(self, element, parent=None):
self._element = element
self._parent = parent
self._children = [] # รองรับลูกได้กี่คนก็ได้
ทำไมต้องใช้ linked structure? เพราะ tree ขยาย/ลดขนาดได้ตลอด เพิ่ม/ลบ node ได้ง่าย ไม่ต้องย้าย memory เหมือน array
ออกสอบ — Position class
Position = wrapper ที่ห่อ node ไว้ ป้องกันไม่ให้แก้ไข node โดยตรง
ทุกครั้งที่ใช้ method ใน tree จะผ่าน _validate(p) ก่อน — ตรวจว่า p เป็น Position ถูกต้อง, อยู่ใน tree นี้จริง, และยังไม่ถูกลบ

method สำคัญ
add_child(p, e) — เพิ่ม node ลูกให้ p, ถ้า p = None แสดงว่ากำลังสร้าง root

def add_child(self, p, e):
parent_node = self._validate(p) if p else None
new_node = self.Node(e, parent_node)
if parent_node:
parent_node._children.append(new_node)
else:
self._root = new_node # ถ้า p=None → สร้าง root
self._size += 1
return self._make_position(new_node)
